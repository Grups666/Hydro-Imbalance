# Spatial Research Foundation - 构建原则

## 项目定位

这是一个**通用空间研究基座**，而非特定领域的应用。

- **Foundation（基座）**: 提供地图渲染、图层管理、交互控制、面板布局等通用能力
- **Module（模块）**: 提供特定领域的数据、图层、面板、逻辑

基座不知道"水文"、"气候"、"生态"是什么，只知道"图层"、"要素"、"面板"。

## 核心原则

### 1. 结构化而非缝补

**禁止**: 在已有代码上打补丁、堆砌功能、绕过问题

**要求**: 发现问题后，理解根本原因，重构相关部分，删除无效代码

示例：
- 点击偏移问题 → 不是调整偏移量参数，而是检查坐标系统是否正确
- 面板显示异常 → 不是添加额外样式覆盖，而是检查布局结构是否合理

### 2. 模块化而非耦合

**禁止**: 在基座代码中写入领域特定逻辑（如水文模式、流域分类）

**要求**: 领域逻辑全部放入模块，通过声明式接口注册

```javascript
// 基座代码禁止出现
const hydrologicalModes = { snow: '...', monsoon: '...' };
function classifyBasin(basin) { /* 水文规则 */ }

// 模块代码可以出现
class HydrologyModule {
  getLayerIds() { return ['hydro-basins']; }
  render(ctx, viewport) { /* 渲染流域 */ }
  hitTest(lon, lat) { /* 点击检测 */ }
  showInspector(feature) { /* 显示属性面板 */ }
}
```

### 3. 数据与渲染分离

**禁止**: 硬编码数据（如 `window.RESEARCH_EXPLORER`）在渲染逻辑中

**要求**: 模块加载时读取数据，存储在模块实例中

### 4. 事件驱动而非直接调用

**禁止**: 模块直接操作基座 DOM 或调用基座内部方法

**要求**: 通过事件系统通信

```javascript
// 模块触发事件
eventBus.emit('inspector:show', { title, content });

// 基座监听事件
eventBus.on('inspector:show', (data) => { /* 显示面板 */ });
```

### 5. 清晰的生命周期

每个模块必须有明确的生命周期方法：

```javascript
class MyModule {
  async onLoad(app) { /* 加载：注册图层、读取数据 */ }
  onUnload() { /* 卸载：移除图层、清理状态 */ }
  getLayerIds() { /* 返回注册的图层ID */ }
}
```

## 常见问题处理方式

### 问题：点击位置不准确

1. 检查 `screenToGeo` 转换是否正确
2. 检查 Canvas 尺寸是否与实际显示区域一致
3. 检查 `resize()` 是否在正确时机调用
4. **删除**任何手动偏移调整代码

### 问题：面板显示异常

1. 检查 CSS 结构是否合理（flexbox/grid）
2. 检查 JavaScript 是否正确操作 DOM
3. **删除**任何覆盖样式或临时修复代码

### 问题：模块加载后无响应

1. 检查模块是否正确注册图层
2. 检查图层是否被标记为 `interactive: true`
3. 检查 `hitTest` 函数是否正确实现

## 代码审查标准

提交代码前检查：

1. **基座代码中没有领域词汇**（如 hydrology, basin, water cycle）
2. **模块代码中没有基座内部调用**（如直接操作 DOM）
3. **没有注释掉的调试代码**
4. **没有"临时修复"、"TODO"、"FIXME"注释**
5. **每个函数有单一职责**
6. **没有重复的功能代码**

## 问题解决验证标准

**每次修复问题后必须验证，不能假设已解决**：

1. **必须实际测试**：打开浏览器，执行操作，确认结果
2. **不能假设"应该可以"**：代码看起来对不代表实际工作
3. **检查完整流程**：不只是检查修改的代码，而是检查整个交互链路

常见需要验证的场景：

| 问题类型 | 验证方法 |
|---------|---------|
| 点击坐标偏移 | 在地图不同位置点击，确认选中正确的要素 |
| UI 元素不可见 | 刷新页面，实际查看元素是否在预期位置 |
| 面板折叠异常 | 点击隐藏，检查按钮是否消失；点击显示，检查面板是否恢复 |
| 事件未触发 | 在控制台打印日志，确认事件确实被触发 |

**验证失败时的处理**：

1. 不要继续添加代码"修复"
2. 回到原点，理解根本原因
3. 删除无效修改，重新实现
