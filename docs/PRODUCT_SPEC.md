# Spatial Research Operating System 产品规格

本文档定义产品功能需求、用户场景和交互设计。

## 1. 产品愿景

**一句话定义**: 从地图出发的科研知识探索系统。

**核心价值**: 将文献、数据、模型、写作整合到统一的空间上下文中，让研究者从"选择区域"开始，而非从关键词开始。

**差异化定位**:
- 不是文献检索工具（Google Scholar）
- 不是数据平台（Zenodo）
- 不是 GIS 软件（ArcGIS）
- 不是 AI 助手（ChatGPT）
- 是连接这些工具的空间研究操作系统

### 2.1 核心用户

| 用户类型 | 需求 | 使用场景 |
|----------|------|----------|

## 3. 功能规格

### 3.1 地图浏览

**功能描述**: 全屏世界地图，支持流域选择和区域浏览。

**交互规范**:
- 默认视图：全球视野
- 缩放方式：鼠标滚轮 / 双击放大
- 平移方式：鼠标拖拽
- 流域高亮：鼠标悬停显示边界
- 流域选择：点击进入区域详情

**数据源**:
- 陆地轮廓：Natural Earth
- 流域边界：HydroBASINS
- 重点区域：预配置知识包

### 3.2 区域档案

**功能描述**: 显示选中区域的水文模式、研究摘要和关键信息。

**内容结构**:
```
┌─────────────────────────────────┐
│ 区域名称                          │
│ 水文模式标签                      │
├─────────────────────────────────┤
│ 研究摘要                          │
│ [2-3 句核心描述]                  │
├─────────────────────────────────┤
│ 水文循环特征                      │
│ • 循环特征 1                      │
│ • 循环特征 2                      │
│ • 循环特征 3                      │
├─────────────────────────────────┤
│ 空间格局                          │
│ • 格局特征 1                      │
│ • 格局特征 2                      │
└─────────────────────────────────┘
```

### 3.3 文献聚合

**功能描述**: 显示与区域相关的文献列表，支持查看详情。

**列表字段**:
- 标题
- 作者（缩写）
- 年份
- 期刊
- 标签（如有）

**详情弹窗**:
```
┌─────────────────────────────────────┐
│ 文献标题                             │
│ 完整作者列表                         │
│ 年份 | 期刊 | DOI                    │
├─────────────────────────────────────┤
│ 摘要                                 │
│ [摘要内容]                           │
├─────────────────────────────────────┤
│ 本地 PDF │ DOI 链接 │ 引用格式      │
└─────────────────────────────────────┘
```

### 3.4 AI 综述生成

**功能描述**: 基于区域上下文生成研究综述。

**输入**:
- 区域档案（名称、水文模式、研究摘要）
- 关联文献元数据
- 用户问题（可选）

**输出**:
- 结构化综述（中文/英文）
- 引用标注（可点击跳转）
- 不确定性表达

**质量要求**:
- 引用准确：每条引用对应真实文献
- 结构清晰：分节、有逻辑
- 可追溯：结论可追溯到来源

### 3.5 数据集入口（规划中）

**功能描述**: 显示与区域相关的数据集列表。

**列表字段**:
- 数据集名称
- 提供方
- 时间范围
- 空间分辨率
- 访问方式

**详情内容**:
- 变量说明
- 下载链接 / API 入口
- 引用格式
- 使用许可

### 3.6 报告导出（规划中）

**功能描述**: 将区域研究上下文导出为文档。

**导出格式**:
- Markdown
- PDF
- LaTeX（规划中）

**导出内容**:
- 区域档案
- 文献列表
- AI 综述
- 引用列表

## 4. 用户场景

### 4.1 场景一：新课题调研

**用户**: 水文研究者，准备开展华北平原地下水研究

**流程**:
1. 打开系统，地图默认显示全球
2. 定位到华北平原区域
3. 点击流域，查看区域档案
4. 浏览水文模式和空间格局
5. 查看关联文献列表
6. 点击关键文献，阅读摘要
7. 使用 AI 综述功能，生成区域研究背景
8. 导出综述草稿，用于进一步修改

**预期效果**:
- 30 分钟内完成区域背景调研
- 获得结构化知识框架
- 掌握核心文献入口

### 4.2 场景二：研究生学习

**用户**: 研究生，学习全球不同流域的水文特征

**流程**:
1. 浏览全球地图
2. 选择感兴趣的区域
3. 阅读区域档案，理解水文模式
4. 比较不同区域的模式差异
5. 阅读关联文献，深入了解

**预期效果**:
- 系统化学习区域水文
- 建立空间对比视角
- 快速定位关键文献

### 4.3 场景三：项目申请背景撰写

**用户**: 研究者，撰写项目申请书背景部分

**流程**:
1. 选择研究区域（如 Indus 流域）
2. 查看区域档案和文献
3. 生成 AI 综述
4. 导出 Markdown 文档
5. 在写作工具中修改完善

**预期效果**:
- 快速生成背景草稿
- 引用准确可追溯
- 节省文献整理时间

## 7. 非功能需求

### 7.1 性能

- 页面首次加载 < 3 秒
- 地图交互响应 < 100ms
- AI 综述生成 < 30 秒
- 支持 10+ 并发用户（MVP 阶段）

### 7.2 可用性

- 界面简洁，无需培训即可使用
- 关键操作有提示反馈
- 错误信息清晰易懂

### 7.3 兼容性

- 浏览器：Chrome、Firefox、Safari、Edge 最新两个版本
- 设备：桌面优先，平板可用
- 语音：支持 Web Speech API 的浏览器

### 7.4 可维护性

- 代码结构清晰
- 配置外部化
- 日志可追溯
