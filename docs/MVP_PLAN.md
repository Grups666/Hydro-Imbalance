# MVP 计划：Hydrology-first Spatial Research OS

本文档定义第一阶段 MVP 的目标、范围、功能规划和成功标准。

## 1. MVP 定位

**核心假设**: 研究者点击一个流域后，比使用分散工具（Google Scholar、数据门户、GIS、AI 助手）更快获得可信、可追溯、可写作的研究上下文。

**目标用户**:
- 水文与水资源研究者
- 研究生（学习区域水文背景）
- 项目申请人（快速生成研究背景）

**时间窗口**: 3 个月（2025-06 至 2025-08）

---

## 2. 功能范围

### 2.1 核心功能（必须有）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 地图浏览 | 全屏世界地图、缩放、平移 | P0 |
| 流域选择 | 点击流域、显示边界 | P0 |
| 区域档案 | 显示水文模式、研究摘要 | P0 |
| 文献聚合 | 关联文献列表、详情弹窗 | P0 |
| AI 综述 | 生成区域研究综述 | P0 |
| 引用追溯 | 综述内容链接到文献来源 | P0 |

### 2.2 增强功能（应该有）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 数据集入口 | 关联数据集、下载链接 | P1 |
| 报告导出 | Markdown/PDF 导出 | P1 |
| 文献搜索 | 关键词搜索、筛选 | P1 |
| 多语言 | 中英文界面切换 | P2 |

### 2.3 延后功能（不做）

- 用户账户系统
- 模型执行平台
- 实时文献监测
- 全球所有流域高覆盖
- 多学科领域扩展

---

## 3. 知识包目标

### 3.1 流域覆盖

MVP 目标：**50 个重点流域**

当前已有：11 个

下一阶段优先：
1. **亚洲**: 珠江、长江、雅鲁藏布江、塔里木河、阿姆河、锡尔河
2. **欧洲**: 莱茵河、多瑙河、伏尔加河
3. **非洲**: 刚果河、赞比西河、尼日尔河
4. **美洲**: 亚马逊河、密西西比河、科罗拉多河、拉普拉塔河
5. **大洋洲**: 墨累-达令盆地扩展

### 3.2 文献覆盖

MVP 目标：**500 篇关联文献**

当前已有：12 篇核心文献

下一阶段策略：
- 每个流域 5-10 篇代表性文献
- 覆盖不同研究主题（地下水、地表水、气候影响、人类用水）
- 包含综述、观测研究、模型研究

### 3.3 数据集入口

MVP 目标：**50 个关键数据集**

分类：
- 遥感数据（GRACE、MODIS、Landsat、Sentinel）
- 再分析数据（ERA5、GLDAS、MERRA2）
- 站点观测（GRDC、FAO、USGS）
- 模型输出（CMIP6、ISIMIP）
- 社会经济（FAO AQUASTAT、World Bank）

## 5. 成功标准

MVP 成功的判断标准：

### 5.1 功能完整性

- [ ] 研究者可从地图选择任意已配置流域
- [ ] 系统自动显示区域水文模式
- [ ] 系统聚合相关文献并提供详情
- [ ] AI 生成可追溯引用的区域综述
- [ ] 综述可导出为 Markdown 或 PDF

### 5.2 体验对比

| 维度 | 传统工具链 | Spatial Research OS MVP |
|------|------------|-------------------------|
| 获取区域背景 | 搜索多篇论文、手动整理 | 一键查看档案 |
| 发现文献 | Google Scholar 关键词 | 自动关联流域 |
| 发现数据 | 多平台搜索 | 数据入口聚合 |
| 生成综述 | 手动阅读、写作 | AI 辅助生成 |
| 追溯来源 | 手动维护引用 | 自动链接文献 |

### 5.3 质量指标

- 文献元数据准确率 > 95%
- 流域档案覆盖核心水文问题
- AI 综述引用准确可追溯
- 页面加载 < 3 秒
- 无关键 bug

### 5.4 用户验证

- 至少 3 位水文研究者完成测试流程
- 获得"比现有工作流更快/更便捷"的反馈
- 收集改进建议 ≥ 10 条

## 6. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 文献元数据获取困难 | 延迟知识库扩展 | 人工维护种子文献 + OpenAlex 补充 |
| AI 综述质量不稳定 | 用户体验下降 | 模板约束 + 人工审核示例 |
| 数据集链接失效 | 功能不可用 | 定期检查 + 多源备份 |
| 用户测试参与不足 | 验证不充分 | 提前预约 + 激励机制 |
